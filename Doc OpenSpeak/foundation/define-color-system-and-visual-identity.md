
## **Estado**

Propuesto

---

# **Objetivo**

Definir el sistema cromático base de OpoSpeak y los principios que guían las decisiones visuales del producto.

Este documento establece la identidad visual fundamental de la aplicación.

No define componentes concretos ni implementaciones técnicas.

---

# **Principio fundamental**

OpoSpeak no es una academia.

OpoSpeak no es una red social.

OpoSpeak no es una aplicación jurídica.

OpoSpeak es una herramienta de entrenamiento oral a largo plazo.

La identidad visual debe transmitir esa idea.

---

# **Problema**

Muchas aplicaciones para oposiciones terminan pareciéndose a:

- plataformas educativas
- herramientas de productividad
- software corporativo
- aplicaciones jurídicas

Ninguna de esas categorías representa adecuadamente la propuesta de valor de OpoSpeak.

---

# **Objetivo emocional**

La aplicación debe transmitir:

- concentración
- claridad
- disciplina
- progreso
- confianza
- estabilidad

---

## **Debe evitar transmitir**

- presión
- competición
- urgencia
- gamificación
- burocracia
- complejidad técnica

---

# **Concepto visual**

## **Inspiración**

La identidad visual de OpoSpeak se inspira en:

- cuadernos de estudio premium
- tinta de escritura
- editoriales de calidad
- lectura profunda
- trabajo intelectual sostenido

---

## **No se inspira en**

- dashboards corporativos
- aplicaciones financieras
- software jurídico
- videojuegos educativos
- redes sociales

# **Filosofía cromática**

## **Principio**

El color debe acompañar.

No dominar.

---

## **Evitar**

- colores saturados
- contrastes agresivos
- efectos llamativos
- interfaces ruidosas

---

## **Buscar**

- calma visual
- lectura cómoda
- sesiones largas de uso

---

# **Accesibilidad**

## **Requisito**

Toda combinación cromática debe cumplir los criterios definidos en:

- WCAG
- Apple Human Interface Guidelines

---

## **Principio**

La accesibilidad tiene prioridad sobre la estética.

---

# **Uso del color**

## **Información crítica**

Nunca debe depender únicamente del color.

---

## **Ejemplo incorrecto**

Solo rojo y verde.

---

## **Ejemplo correcto**

Color + texto + iconografía.

# **Consistencia**

Toda nueva funcionalidad debe utilizar este sistema cromático.

No deben introducirse nuevos colores de marca sin una decisión explícita.

---

# **Criterio de aceptación**

Una pantalla nueva cumple este documento cuando:

- utiliza la paleta oficial
- mantiene coherencia visual
- respeta accesibilidad
- evita saturación cromática

---

# **Resultado esperado**

Cuando un usuario abra OpoSpeak debe percibir una herramienta seria, tranquila y diseñada para acompañar años de preparación.

La identidad visual debe recordar a un cuaderno de entrenamiento personal.

No a un software corporativo.

No a una plataforma educativa.

No a una red social.

Mi única duda antes de congelar esta spec es si queremos que OpoSpeak y Liroa compartan cierta “familia visual” (mismo Warm Sand y filosofía editorial) o si prefieres que sean dos marcas completamente independientes. Personalmente, mantendría una familia visual reconocible, porque ambos productos comparten muchas ideas: privacidad, reflexión, largo plazo y ausencia de gamificación.
